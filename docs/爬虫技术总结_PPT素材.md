### 第2页：技术难题攻克（重点页）

**标题**：爬虫开发中的关键技术挑战

**表格呈现**：

| 挑战 | 现象 | 解决方案 |
|------|------|---------|
| 🔴 翻页缓存 | 82K条中93.7%重复 | 防缓存头+URL时间戳+内容校验 |
| 🔴 CSS失效 | 价格全部为0 | 逆向新版CSS，多级选择器回退 |

**配图建议**：翻页缓存问题的"修复前 vs 修复后"对比图（见截图指南第2项）

**一句话总结**：遇到了翻页缓存、CSS改版、IP封禁、链家反爬四大难题，通过三层防缓存体系、多级选择器回退、代理池自动刷新等方案逐一攻克。

### 第3页：代理方案演进

**标题**：代理IP方案的三次迭代

**流程图**：
```
Cookie直爬 → 多线程被封
    ↓
免费代理(失败) → GitHub开源(失败)
    ↓
快代理隧道代理 → 82K条但93.7%重复
    ↓
快代理私密代理池 → 每条请求独立IP → ✅ 当前方案
```

**关键数据**：
- 私密代理池：20个IP，API动态提取，自动刷新
- 页间延迟：5-8秒，每IP请求密度低
- 安居客38区县，链家12区县

**一句话总结**：经历了Cookie直爬被封、免费代理不可用、隧道代理翻页缓存的试错过程，最终采用私密代理池方案，实现了稳定、高效的数据采集。

### 第5页（可选）：技术亮点

**标题**：爬虫技术亮点

**要点**：
1. **三层防缓存体系**：HTTP头 → URL时间戳 → 内容ID校验
2. **多级选择器回退**：`.content-price` → `.price-wrap` → `.price` → 标题正则
3. **代理池自动管理**：快代理API → 本地缓存 → 120秒冷却 → 失败自动刷新
4. **断点续爬 + CSV合并**：区县级断点，重启不丢数据
5. **早停优化**：连续3页无新增自动停止，节约80%代理配额
