## 1. 功能说明（200-300字）

本模块实现了重庆市二手房源数据的自动采集，采用双数据源互补策略：以**安居客**（m.anjuke.com）为主力数据源，**链家**（cq.lianjia.com）为辅助数据源。安居客移动站每页展示60-71条房源信息，反爬门槛较低，适合大规模采集；链家每页30条，反爬机制较为严格，用于补充核心区域数据。

技术栈采用 Python + Requests + BeautifulSoup。针对安居客设计了三文件架构：`anjuke_fast.py`（多线程+隧道代理，纯列表页快速采集82K条）、`anjuke_detail.py`（单线程进详情页，获取全部22字段）、`anjuke_backfill.py`（读CSV按标题匹配补坐标等缺失字段）。链家为 `lianjia_spider.py`（列表页直解析，每区获取30条）。

采集流程为：遍历重庆市38区县 → 按区县翻页爬取移动站列表页 → 可选进入详情页补坐标/装修等字段 → 结构化存储为CSV → MD5指纹去重。翻页时采用随机延迟、Referer伪装和Connection:close强制换IP策略。截至采集完成，共获取重庆市30区县、**82,241条**二手房数据。

## 2. 核心代码片段

### 2.1 安居客移动站列表页解析（anjuke_fast.py）

```python
def parse_list_page(html):
    """从移动站列表页HTML提取房源基本信息（不进详情页）"""
    soup = BeautifulSoup(html, 'lxml')
    items = soup.select('li.item-wrap')    # 移动站房源卡片
    results = []

    for item in items:
        # 房源ID: 从<a href="/cq/sale/S2756841929583619/">提取
        href = item.find('a').get('href', '')
        hid = re.search(r'/S(\d+)/', href).group(1)

        # 标题: <span class="content-title">渝中区 和泓江山国际 三房 126万</span>
        title = item.select_one('.content-title').get_text(strip=True)

        # 描述字段: 3室2厅 / 91.77㎡ / 北 / 渝中 / 大坪
        descs = item.select('.content-desc')
        layout = descs[0].get_text(strip=True)     # "3室2厅"
        area = float(descs[1].get_text(strip=True).replace('㎡',''))
        orientation = descs[2].get_text(strip=True) # "北"

        # 总价: <span class="price">126万</span>
        price_text = item.select_one('.price').get_text(strip=True)
        total_price = float(re.sub(r'[^\d.]', '', price_text))

        # 小区名: 从<img alt="和泓江山国际3室2厅...">提取
        img_alt = item.find('img').get('alt', '')
        community = re.match(r'(.+?)\d+室', img_alt).group(1)

        results.append({
            'id': hid, 'title': title, 'total_price': total_price,
            'area': area, 'layout': layout, 'community': community,
            'orientation': orientation, 'source': 'anjuke'
        })
    return results
```

### 2.2 安居客详情页坐标获取（anjuke_detail.py / anjuke_backfill.py）

```python
def fetch_detail(hid):
    """从桌面站+移动站详情页获取全字段"""
    # --- 桌面站: 经纬度 ---
    # <meta name="location" content="province=重庆;coord=106.52,29.54">
    resp = requests.get(f'https://chongqing.anjuke.com/prop/view/S{hid}',
                        headers={'User-Agent': UA_DESKTOP}, proxies=PROXIES)
    coord = re.search(r'coord=([\d.]+),([\d.]+)', resp.text)
    lng, lat = float(coord.group(1)), float(coord.group(2))

    # --- 移动站: 装修/年代/楼层 ---
    # <meta name="keywords" content="126万,3室2厅1卫,91.77平米,
    #        13730元/平米,2017年,北,精装修,普通住宅">
    resp2 = requests.get(f'https://m.anjuke.com/cq/sale/S{hid}/',
                         headers={'User-Agent': UA_MOBILE}, proxies=PROXIES)
    kw = re.search(r'<meta[^>]+name="keywords"[^>]+content="([^"]+)"', resp2.text)
    content = kw.group(1)
    bathrooms = re.search(r'(\d+)室\d+厅(\d+)卫', content).group(2)
    unit_price = re.search(r'([\d.]+)元/平米', content).group(1)
    build_year = re.search(r'(\d{4})年', content).group(1)
    decoration = re.search(r'(精装修|简装|毛坯)', content).group(1)

    return {'lng': lng, 'lat': lat, 'bathrooms': int(bathrooms),
            'unit_price': float(unit_price), 'build_year': int(build_year),
            'decoration': decoration}
```

### 2.4 数据去重指纹生成

```python
def fingerprint(community, district, area, rooms, total_price):
    """生成房源唯一指纹，用于跨区县去重"""
    key = f"{community}|{district}|{area}|{rooms}|{total_price}"
    return hashlib.md5(key.encode()).hexdigest()
```

## 3. 技术要点/难点

### 难点1：安居客移动站 vs 桌面站的选择

**问题**：桌面站(chongqing.anjuke.com)2026年起全面SPA化，requests无法获取房源数据（返回空壳HTML）。移动站(m.anjuke.com)仍为服务端渲染，HTML中直接包含房源数据。

**解决**：全部切换为移动站URL。移动站列表页每页60-71条，选择器为`li.item-wrap`，完全满足采集需求。

### 难点2：IP封禁与隧道代理

**问题**：安居客和链家均对高频请求触发IP级别封禁。安居客使用网易易盾验证码，链家使用贝壳反爬系统。

**解决**：
1. 使用快代理隧道代理（每次请求`Connection: close`自动切换IP）
2. 移动站列表页请求间隔10-20秒
3. 实现`smarter_get`策略：先直连，失败后走代理兜底
4. 多区县并行（6线程）降低单区县请求密度

### 难点3：字段不完整与多文件架构

**问题**：移动站列表页天然缺少卫生间数、单价、装修、年代、楼层、经纬度。进详情页才能获取，但详情页请求量是列表页的60倍，大幅降低速度。

**解决**：设计三文件架构——
- `anjuke_fast.py`：不进详情页，快速覆盖30区县，unit_price和bathrooms通过计算兜底
- `anjuke_detail.py`：进详情页获取全字段，适合小规模精确采集
- `anjuke_backfill.py`：读fast产出的CSV，标题匹配后进详情页补字段

### 难点4：远郊区县数据缺失

**问题**：云阳、忠县、巫溪等极偏远区县，代理IP频繁被拦截（IncompleteRead），15次重试全部失败。

**解决**：目前30/38区县已覆盖。偏远区县建议换IP（手机热点）后单独运行。
