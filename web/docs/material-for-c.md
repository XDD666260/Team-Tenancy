## 三、技术选型概述（用于第6章开头）

Web 前端采用 **Next.js 15 + React 19 + TypeScript + Tailwind CSS** 技术栈，配合 **Recharts** 交互式图表库和 **GSAP** 专业动画引擎。平台部署于 Netlify，采用静态导出模式，无需服务器。

| 技术 | 用途 |
|------|------|
| Next.js 15 | 前端框架，App Router 路由，静态导出 |
| TypeScript | 类型安全的开发语言 |
| Tailwind CSS | 原子化 CSS，配合 Design Token 统一配色 |
| Recharts | 声明式图表库（柱状图、雷达图、散点图、环形图） |
| GSAP ScrollTrigger | 滚动触发动画、入场淡入效果 |

### 4.3 房价预测分析（图6-3）

**功能**：4 个机器学习模型（RF/GB × 总价/单价）的评估指标 + 特征重要性对比。

**模型性能数据**：

| 模型 | 预测目标 | 测试R² | 测试MAE | 5折CV R² |
|------|---------|--------|---------|----------|
| RandomForest | 总价(万) | 0.5345 | 32.83万 | 0.508 ± 0.02 |
| RandomForest | 单价(元/㎡) | 0.4023 | 3267元/㎡ | 0.391 ± 0.02 |
| GradientBoosting | 总价(万) | 0.5137 | 33.03万 | 0.488 ± 0.03 |
| GradientBoosting | 单价(元/㎡) | 0.3797 | 3283元/㎡ | 0.369 ± 0.03 |

**特征重要性 TOP 5**：

| 排名 | 特征 | 重要性 | 解读 |
|------|------|--------|------|
| 1 | 面积 | 33.7% | 最大影响因素 |
| 2 | 小区均价编码 | 25.5% | 小区档次决定房价 |
| 3 | 区县均价编码 | 25.5% | 地段是核心 |
| 4 | 户均面积 | 6.9% | 户型合理性 |
| 5 | 总楼层 | 4.3% | 高层vs低层价差 |

**核心发现**：面积 + 地段（小区/区县）解释房价变化的 84.7%。

### 4.5 关联规则挖掘（图6-5）

**功能**：Apriori 算法 50 条关联规则可视化（热力图 + TOP 10 表格）。

**TOP 5 规则**：

| # | 前提条件 | 结论 | 支持度 | 置信度 | 提升度 |
|---|---------|------|--------|--------|--------|
| 1 | 单价12000-18000 + 面积90-120㎡ | 两江新区 + 总价120-200万 | 2.3% | 53% | 7.33 |
| 2 | 单价12000-18000 + 面积90-120㎡ | 总价120-200万 + 3室 | 2.4% | 56% | 7.12 |
| 3 | 单价8000-12000 + 总价<50万 | 面积<60㎡ | 2.4% | 99% | 6.82 |
| 4 | 区县=武隆区 + 总价<50万 | 南向 + 面积<60㎡ | 2.7% | 66% | 6.72 |
| 5 | 单价12000-18000 + 面积90-120㎡ | 总价120-200万 | 3.9% | 92% | 6.34 |

**关键发现**：
1. 两江新区是中高价位核心区域（Lift=7.33）
2. 低价必小面积：总价<50万 → 面积<60㎡ 置信度 99%
3. 改善型甜蜜点：90-120㎡ + 三室 + 总价120-200万

## 六、综合结论（用于第7章）

### 6.1 项目成果

成功构建重庆二手房数据全链路系统：数据采集（50,507条）→ 数据清洗 → MySQL存储 → 机器学习分析（RF/GB/KMeans/Apriori）→ FastAPI后端 → **双端可视化（Android App + Web平台）**。

### 6.2 关键数据发现

1. **面积+地段=房价的85%**：面积(33.7%)+小区编码(25.5%)+区县编码(25.5%)
2. **金字塔市场结构**：远郊大户型(36.4%)+紧凑刚需(27.7%)占64%，高端豪宅仅5.6%
3. **两江新区是中高价位核心**：Lift=7.33，远超随机概率
4. **低价必小面积**：总价<50万房源 99%面积<60㎡
5. **改善型甜蜜点**：90-120㎡+三室+总价120-200万

### 6.3 技术亮点

- 双端展示（Android + Web），适应不同答辩场景
- Next.js 静态导出 + Netlify CDN，零服务器成本
- GSAP 苹果级滚动动画 + Recharts 交互式图表
- 50,507 条真实数据驱动，Mock 数据兜底

### 6.4 不足与改进

1. 预测模型 R²(0.53)可提升，引入 XGBoost + 更多特征（学区、地铁）
2. 可接入后端 API 实现数据实时更新
3. 移动端适配可进一步优化
